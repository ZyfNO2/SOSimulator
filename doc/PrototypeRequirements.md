# 原型需求文档

## 1. 当前 MVP 目标

做出一个可玩的 React 桌面式卡牌原型，让玩家可以：

- 从 `精力 / 时间 / 春日0 / 商店 / 天气（晴）` 开局
- 通过拖拽堆叠推进蓝伞主线
- 看到长门介入、天气升级与三条结局分歧
- 在不看开发者工具的前提下，完成至少一条结局线

当前原型是“自由桌面堆叠合成”而不是“动作框 + 卡槽”。

---

## 2. 必备功能

- 显示开始界面，并在点击开始后让初始牌依次显现
- 在桌面上自由拖动卡牌
- 卡牌之间可吸附成父子链
- 父子链满足规则时自动开始倒计时产出
- 完成后生成新牌、保留牌或结局牌
- `精力 / 时间` 母牌可在左下角分离子牌并自动回充
- 右上角显示主线收纳栏，可归档和取回主线卡
- 点击任意卡牌可展开详情弹窗
- 发现蓝伞后，天气每 3 分钟自动上升一级
- 到达 `天气（特大暴雨）` 后，倒计时结束会强制进入 `虚构失主出现`

---

## 3. 当前验收流程

玩家应能完成类似下面的主线推进：

1. `商店 + 精力 -> 商店 + 金钱`
2. `金钱 + 商店 -> 商店 + SOS团活动材料`
3. `春日0 + SOS团活动材料 -> 春日1`
4. `春日1 + 时间 + 精力 -> 春日1 + 失物招领室0`
5. `失物招领室0 + 精力 -> 失物招领室0 + 蓝色雨伞0`
6. `商店 + 蓝色雨伞0 -> 蓝色雨伞1 + 证词`
7. `证词` 首次出现后自动刷出 `长门0`
8. 继续推进至 `春日的执念 / 世界异常 / 不存在的失主 / 结论 / 虚构失主`
9. 最终进入：
   - `阿虚认领蓝色雨伞`
   - `虚构失主出现`
   - `灭世线`

---

## 4. 交互验收

- 合法堆叠时，子牌会吸附到父牌标题横线下方
- 不合法时，卡牌保持自由桌面位置
- 拖动父牌时，整条堆叠链一起移动
- 2 张牌规则允许正反顺序匹配
- 3 张及以上规则中，第一个输入必须是父牌，后续顺序可互换
- 母牌左键拖动时，会分离出 1 张子牌
- 场上同类资源总量小于 `2` 时，母牌出现回充遮罩
- 回充结束后母牌数量加 `1`
- 子牌拖回母牌时，能并回数量
- 点击卡牌后能看见简介与当前可见提示
- 尚未出现过的相关卡，不应提前在提示中剧透

---

## 5. 当前结局验收

- `天气（暴雨） + 世界异常 + 春日3 -> 灭世线`
- `天气（大雨） + 结论 -> 阿虚认领蓝色雨伞`
- `天气（大雨） + 虚构失主 -> 虚构失主出现`
- 若玩家拖太久，`天气（特大暴雨）` 倒计时结束后，场上其他牌全部消失，只留下 `虚构失主出现`

---

## 6. 当前非目标

下面这些内容不属于当前代码已经实现的范围：

- 固定动作框与卡槽系统
- 独立日志面板
- 按天数推进
- 店铺关闭事件
- 朝比奈 / 古泉 的正式卡牌与玩法
- 干扰怪、敌人、人物冷却卡
- 存档 / 读档
- 云同步或后端依赖

---

## 7. 后续扩展方向

- 把更多提示从自动模板改成逐卡手写
- 增加“已见过 / 已触发过”的剧情记录
- 做真正的按天推进与第四天关店
- 加入干扰事件、人物冷却、关底要求
- 补前端可见日志与更完整的结局回顾
