# 原型需求文档

## 1. MVP 目标

做出一个可玩的 React 原型，让玩家能通过拖拽卡牌完成一条胜利路径。

## 2. 必备功能

- 显示初始卡牌。
- 显示动作框和槽位。
- 拖动卡牌到槽位。
- 槽位校验合法性。
- 满足配方后启动动作。
- 动作倒计时。
- 完成后产出卡牌和日志。
- 根据条件解锁新动作或事件。
- 触发一个胜利结局。

## 3. 验收流程

玩家应能完成以下流程：

1. 使用 `精力` 或 `贫瘠线索` 进行探索。
2. 获得 `奇怪线索`。
3. 使用 `研究` 将 `奇怪线索` 转化为 `隐秘知识`。
4. 通过梦境或休息获得 `梦境残响`。
5. 使用 `隐秘知识 + 梦境残响 + 意志` 执行仪式。
6. 获得胜利结局 `窥见晨星`。

## 4. 交互验收

- 合法拖放后卡牌进入槽位。
- 非法拖放被拒绝，并显示反馈。
- 动作运行时输入卡牌不可再次拖动。
- 动作完成后玩家能看见新增卡牌。
- 日志能说明动作结果。
- 胜利结局出现后游戏停止继续动作。

## 5. 非目标

- 不要求美术最终品质。
- 不要求复杂随机事件。
- 不要求账号、云存档、后端。
- 不要求移动端完整体验。
- 不要求复刻任何现成游戏的文本或数值。

## 6. 后续扩展

- 多结局。
- 卡牌寿命。
- 随机事件牌堆。
- 更复杂的仪式系统。
- 存档/读档。
- 音效和环境氛围。
