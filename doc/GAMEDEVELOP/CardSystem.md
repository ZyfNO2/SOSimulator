# 卡牌系统文档

## 1. 卡牌定位

卡牌是游戏中所有资源、状态、知识、线索和剧情物件的统一载体。玩家通过拖动卡牌驱动行动。

## 2. 卡牌类型

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| `resource` | 基础资源 | 精力、金钱、时间 |
| `clue` | 推进探索 | 贫瘠线索、奇怪线索 |
| `knowledge` | 研究产物 | SOS团成立宣言、隐秘知识 |
| `state` | 玩家/世界状态 | SOS（思绪）、疲惫、惊惧 |
| `tool` | 可重复使用道具 | 旧地图、黄铜钥匙 |
| `ritual` | 高阶行动材料 | 梦境残响、意志 |
| `ending` | 结局卡 | 窥见晨星 |
| `location` | 场景地点 | 文艺社活动室、二年级教室 |
| `character` | 人物卡 | Test（人物） |
| `mechanic` | 机制卡 | 校园探索（自动生成地点） |
| `leader` | 团长 | 凉宫春日 |
| `member` | 团员 | 阿虚 |
| `alien` | 外星人 | 长门有希 |
| `traveler` | 未来人 | 朝比奈实玖瑠 |
| `esper` | 超能力者 | 古泉一树 |

## 3. 核心属性

- `id`：定义 id，稳定不变。
- `name`：显示名称。
- `kind`：卡牌类型。
- `kindLabel`：类型显示标签。
- `note`：玩家可见说明。
- `accent`：样式主题标识。
- `isStackable`：是否可堆叠。

## 4. 卡牌实例

卡牌定义描述"这是什么"，卡牌实例描述"桌上这一张是什么"。

- 同一种定义可以有多个实例。
- 可堆叠卡牌可以用 `quantity` 表示数量。
- 不可堆叠卡牌保持独立实例，方便被动作框占用或消耗。

## 5. 拖拽规则

- 卡牌可在桌面上自由拖动。
- 靠近其他卡牌时吸附到标题线（31px 偏移）。
- **吸附前检测**：如果组合无产出规则或 `canProduce` 返回 false，拒绝吸附并弹开。
- 拖拽带动整个后代链一起移动。
- 动作运行中，已放入的输入卡牌被锁定。
- 动作完成后，根据配方决定消耗、返还或转化。

## 6. 槽位规则

槽位只依赖数据判断，不依赖 UI 文案。

示例方向：

- `探索` 第一槽需要 `resource` 或带 `explore` 标签。
- `研究` 第一槽需要 `clue`，第二槽可选 `resource: 精力`。
- `仪式` 需要 `knowledge`、`ritual`、`state: 意志`。

## 7. 产出规则

- 固定产出：必定生成指定卡牌。
- 条件产出：根据 flags 或输入标签生成不同卡牌。
- **产出前检测**：通过 `CanProduceCheck` 回调在转条前验证能否产出。
- 随机产出：MVP 暂不启用，后续可以加权表。
- 剧情产出：触发事件节点或结局。

### CanProduceCheck 机制

```typescript
export type CanProduceCheck = (
  rule: CardOutputRule,
  parentCard: TableCard,
  childCard: TableCard,
  allCards: TableCard[],
) => boolean
```

使用场景：
- **全局唯一卡牌**：如 SOS团成立宣言，已存在时阻止再次产出。
- **条件限制**：如需要特定 flag 才能产出的卡牌。
- **数量限制**：如某卡牌最多同时存在 N 张。

## 8. 特殊机制

### 地点自消耗

- 地点卡生成时启动倒计时（默认15秒）。
- 右上角显示红色进度圈。
- 放入人物后暂停倒计时。
- 人物取出后恢复倒计时。
- 倒计时结束地点卡自动消失。

### 机制卡（无限循环）

- 机制卡不消耗自身。
- 按固定间隔自动执行动作。
- 如「校园探索」每8秒生成一个地点。

## 9. 后续扩展

- 卡牌寿命与倒计时。
- 腐化、污染、升级等状态。
- 卡牌合并和拆分。
- 卡牌右键详情面板。
