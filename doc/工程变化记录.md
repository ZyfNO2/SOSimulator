## 一、已实现的功能

### 1. 核心拖拽与堆叠系统

- 卡牌可在桌面上自由拖动
- 拖拽时自动脱离原父级堆叠链
- 靠近其他卡牌时吸附到标题线（31px 偏移）
- 堆叠关系为单向链式结构（parent → child），非树形
- 拖拽带动整个后代链一起移动
- z-index 自动管理，拖拽卡牌置顶
- **拖拽吸附前检测**：无产出规则或 `canProduce` 返回 false 时，拒绝吸附并弹开

### 2. 生产（配方）系统

- 基于 `CardOutput.json` 的配方规则驱动
- 当 parent + child 卡牌组合匹配规则时，自动触发 `ProductionRun`
- 生产进度以环形动画展示（生长 → 进行 → 收缩）
- 生产完成自动消耗子卡牌（可选）并产出新卡牌
- 支持自动重排队列（`PRODUCTION_AUTO_REQUEUE_LEAD_MS`）
- **产出前检测机制**：`getProductionMatches` 支持传入 `CanProduceCheck` 回调，在匹配后、转条前检测能否产出
- **SOS宣言唯一性**：通过 `canProduceCheck` 确保 SOS团成立宣言 全局唯一

### 3. 关卡系统

- **关卡选择入口**：`LevelSelect` 组件展示可选关卡
- **关卡1：团长的召唤**：
  - 凉宫春日初始不可移动（灰色，点击/拖动震动提示"凉宫春日还没有发言"）
  - 阿虚拖入凉宫春日 → 产出 SOS团成立宣言
  - 宣言产出后凉宫春日解锁（可拖动）
  - 凉宫拖入宣言 → 产出 SOS（思绪）
  - SOS思绪产出后触发校园探索阶段

### 4. 校园探索系统

- **地点生成器**：SOS思绪产出后左上角刷出「校园探索」机制卡
- 每 **8秒** 自动生成一张地点卡（循环6个地点）
- 地点卡自带 **红圈自消耗条**（15秒），转完自动消失
- 人物（凉宫/阿虚）拖入地点卡 → 停止红圈，开始 **5秒** 黑色进度条
- 人物消耗完成 → 产出 **Test（人物）**

### 5. 日志系统

- `LogPanel` 组件显示最近50条活动日志
- 自动滚动到底部
- 日志内容覆盖：生产开始、生产完成、地点发现/消失、角色解锁

### 6. 卡牌生成动画

- 新卡牌生成时带有 spawn 动画（从原点缩放入场，520ms）
- 生产产出、地点生成均使用该动画

## 三、与设计文档（AI_SDD）的差距

### 架构层面

| 设计目标 | 当前实现 | 差距说明 |
|---------|---------|---------|
| `useReducer` + 集中式状态管理 | `useState` × 5 + 3个 `useEffect` | 状态分散，逻辑耦合在 App.tsx 中 |
| `GameProvider` 提供 context | 无 | 所有状态通过 props 传递，无 context/reducer |
| 纯函数规则 + selectors | 部分纯函数在 `game/` 目录 | 但 App.tsx 中仍有大量硬编码规则分支 |
| 动作框（ActionBox）+ 槽位（ActionSlot） | 无 | 当前是桌面堆叠自动触发，无动作框概念 |
| 玩家主动点击"开始"和"领取" | 无 | 配方匹配后自动开始、自动完成 |
| 卡牌 location 状态（table/slot/locked/discard） | 无 | 只有桌面坐标和堆叠链 |
| 事件树与选择系统 | 无 | 只有结局弹窗，无事件节点和分支选择 |
| 存档系统（localStorage） | 无 | 每次刷新重置到初始状态 |
| 单元测试（vitest） | 无 | 未配置测试运行器 |

### 卡牌类型差距

| 设计类型 | 当前类型 | 状态 |
|---------|---------|------|
| resource（资源） | resource | ✓ |
| clue（线索） | clue | ✓ |
| knowledge（知识） | knowledge | ✓（SOS团成立宣言） |
| state（状态） | state | ✓（SOS思绪） |
| tool（道具） | tool | ✓（电影剧本） |
| ritual（仪式） | 无 | ✗ 未实现 |
| ending（结局） | ending | ✓（世界的真相） |
| location（地点） | location | ✓（6个校园地点） |
| character（人物） | character | ✓（Test人物） |
| mechanic（机制） | mechanic | ✓（校园探索生成器） |

### 交互模式差距

**设计目标交互流程：**

```
拖动卡牌 → 放入动作框槽位 → 点击"开始" → 倒计时 → 点击"领取" → 获得结果
```

**当前实际交互流程：**

```
拖动卡牌 → 堆叠到另一张卡牌上 → 自动匹配配方 → 自动开始倒计时 → 自动完成 → 自动产出
```

差距：
- 无动作框和槽位概念
- 无"开始"按钮，匹配即触发
- 无"领取"按钮，完成自动结算
- 非法组合在拖拽时即被拒绝（弹开）

### 配方系统差距

| 设计目标 | 当前实现 |
|---------|---------|
| 多槽位配方（如仪式需要3种卡牌） | 仅支持 parent + child 两卡配对 |
| 按标签（tags）匹配 | 仅按 definitionId 精确匹配 |
| 按类型（type）匹配 | 无 |
| 配方优先级选择 | 无，直接匹配第一条规则 |
| 配方消耗模式：consume / return / transform | 仅支持 consume（布尔值） |
| 配方产出可指定数量和位置 | 仅支持产出1张到桌面 |
| **产出前条件检测** | **✓ 已实现 `CanProduceCheck`** |

## 五、新机制详解

### 产出前检测（CanProduceCheck）

```typescript
// production.ts
export type CanProduceCheck = (
  rule: CardOutputRule,
  parentCard: TableCard,
  childCard: TableCard,
  allCards: TableCard[],
) => boolean

export function getProductionMatches(
  cards: TableCard[],
  canProduce?: CanProduceCheck,
)
```

使用场景：
- **SOS宣言唯一性**：已存在宣言时，`haruhi` + `kyon` 组合被视为不匹配
- **拖拽吸附验证**：`handlePointerMove` 中临时构建叠放状态，调用 `getProductionMatches` 验证是否有效

### 地点自消耗系统

- 地点卡生成时启动 15秒 倒计时
- 右上角显示红色进度圈
- 放入人物后暂停倒计时（进度圈停止）
- 人物取出后恢复倒计时
- 倒计时结束地点卡自动消失

### 地点生成器

- 机制卡不消耗自身，无限循环
- 按固定顺序轮换6个地点
- 生成位置以桌面中心为基准，黄金角螺旋分布
