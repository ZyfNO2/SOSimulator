# GDD：拖拽卡牌秘仪原型

## 1. 游戏愿景

做一个受《密教模拟器》启发的网页卡牌桌面原型：玩家通过拖动卡牌到动作框槽位中，组合资源、线索和意志，逐步揭开世界真相并达成一个胜利结局。

## 2. 目标体验

- 像整理桌面一样操作卡牌。
- 每次行动都由“卡牌放入槽位 -> 开始倒计时 -> 产出结果”组成。
- 玩家通过观察卡牌标签、日志文本和事件提示推理下一步。
- 氛围偏神秘、克制、渐进，而不是即时战斗。

## 3. 核心循环

1. 获得基础卡牌：精力、金钱、线索、梦境、碎片知识。
2. 将卡牌拖入动作框：探索、研究、工作、休息、仪式。
3. 动作框校验槽位和配方。
4. 计时结束后消耗输入并产出新卡牌、日志或剧情标记。
5. 解锁新动作、新卡牌或新事件。
6. 重复循环，直到触发胜利或失败。

## 4. MVP 胜利路径

- 初始：玩家拥有 `精力`、`贫瘠线索`、`日常工作`。
- 第一阶段：通过 `工作` 获得 `金钱`，通过 `探索` 获得 `奇怪线索`。
- 第二阶段：通过 `研究` 将线索转化为 `隐秘知识`。
- 第三阶段：通过 `仪式` 使用 `隐秘知识 + 意志 + 梦境残响` 触发 `门后之光`。
- 胜利：获得结局 `窥见晨星`。

## 5. 失败方向

- 资源枯竭：长时间没有金钱或精力。
- 疯狂累积：过度研究禁忌知识导致理智崩溃。
- 时间压力：某些威胁事件未处理。

MVP 可以只做胜利结局，失败只保留数据和文案占位。

## 6. 原型范围

### 必做

- 卡牌拖拽。
- 动作框槽位校验。
- 动作倒计时。
- 输入消耗与输出产出。
- 日志反馈。
- 一个完整胜利结局。

### 暂不做

- 大量剧情文本。
- 复杂经济系统。
- 多周目继承。
- 商店、战斗、角色养成。
- 网络同步。

## 7. 内容基调

- 中文文本简短、暗示性强。
- 卡牌名称偏抽象，但功能要清楚。
- 日志承担提示作用，避免让玩家完全猜谜。
