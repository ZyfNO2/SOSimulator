# 卡牌系统文档

## 1. 卡牌定位

卡牌是游戏中所有资源、状态、知识、线索和剧情物件的统一载体。玩家通过拖动卡牌驱动行动。

## 2. 卡牌类型

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| `resource` | 基础资源 | 精力、金钱、时间 |
| `clue` | 推进探索 | 贫瘠线索、奇怪线索 |
| `knowledge` | 研究产物 | 隐秘知识、门之铭文 |
| `state` | 玩家状态 | 疲惫、惊惧、清醒 |
| `tool` | 可重复使用道具 | 旧地图、黄铜钥匙 |
| `ritual` | 高阶行动材料 | 梦境残响、意志 |
| `ending` | 结局卡 | 窥见晨星 |

## 3. 核心属性

- `id`：定义 id，稳定不变。
- `name`：显示名称。
- `type`：卡牌类型。
- `tags`：规则匹配标签，如 `mental`、`money`、`forbidden`。
- `description`：玩家可见说明。
- `stackable`：是否可堆叠。
- `unique`：是否全局唯一。
- `visibility`：默认可见、解锁后可见或隐藏。

## 4. 卡牌实例

卡牌定义描述“这是什么”，卡牌实例描述“桌上这一张是什么”。

- 同一种定义可以有多个实例。
- 可堆叠卡牌可以用 `quantity` 表示数量。
- 不可堆叠卡牌保持独立实例，方便被动作框占用或消耗。

## 5. 拖拽规则

- 卡牌可从手牌/桌面区拖入动作框槽位。
- 槽位按类型、标签、数量校验。
- 不合法槽位应有明确视觉反馈。
- 动作运行中，已放入的输入卡牌被锁定。
- 动作完成后，根据配方决定消耗、返还或转化。

## 6. 槽位规则

槽位只依赖数据判断，不依赖 UI 文案。

示例方向：

- `探索` 第一槽需要 `resource` 或带 `explore` 标签。
- `研究` 第一槽需要 `clue`，第二槽可选 `resource: 精力`。
- `仪式` 需要 `knowledge`、`ritual`、`state: 意志`。

## 7. 产出规则

- 固定产出：必定生成指定卡牌。
- 条件产出：根据 flags 或输入标签生成不同卡牌。
- 随机产出：MVP 暂不启用，后续可以加权表。
- 剧情产出：触发事件节点或结局。

## 8. 后续扩展

- 卡牌寿命与倒计时。
- 腐化、污染、升级等状态。
- 卡牌合并和拆分。
- 卡牌右键详情面板。
